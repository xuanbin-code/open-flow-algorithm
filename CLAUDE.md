## Project Overview

FlowgorithmJS is a third-party tool that reads `.fprg` files (Flowgorithm's XML flowchart format) and renders them as SVG in a browser. The repository contains both the core `flowgorithm.js` library and a Vue 3 demo app in `demo-f-g/`.

The `demo-f-g/` project is now also a **Tauri v2 desktop application** тАФ it can run as a native Windows/macOS/Linux app with access to system APIs (file system, shell, etc.).

## Key Commands

### Core Library (root directory)
- No build step required тАФ `flowgorithm.js` is a vanilla JS library
- Open `drawLocal.html` or `drawWeb.html` directly in a browser to test

### Vue Demo App (`demo-f-g/`)
```bash
cd demo-f-g
pnpm install       # Install dependencies (uses pnpm)
pnpm dev           # Start dev server (Vite) тАФ browser only

# Tauri desktop app

```

> **Windows prerequisite:** Rust toolchain required (`rustup`). WebView2 is built into Windows 11.

## Architecture

### Core Library (`flowgorithm.js`)
The library parses Flowgorithm's XML format and renders SVG flowcharts. Key entry points:
- `drawFlowchartFromUrl(url, selector, options)` тАФ fetch and render a .fprg file
- `drawFlowchartFromSource(xmlString, selector, options)` тАФ render from XML string
- `drawFlowchart($xml, selector, options)` тАФ core rendering function

**Supported XML elements:** `declare`, `output`, `input`, `assign`, `call`, `if`, `for`, `while`, `do`, `more`

**Options:** `aH` (arrow length), `aT` (arrow tip size), `groupInput`, `viewDesc`, `itMode`, `labelTrue`, `labelFalse`

### Vue Demo (`demo-f-g/src/`)
A Vue 3 + TypeScript app that parses `.fprg` XML and renders an interactive flowchart using VueFlow.

**Entry:** `main.ts` тЖТ `App.vue` тЖТ `sandbox/index.vue`

**Key files:**
- `sandbox/index.vue` тАФ Main flowchart UI (VueFlow canvas, edge-click insertion, layout debug panel)
- `sandbox/flowchart-engine.ts` тАФ Core engine: AST тЖТ VueFlow nodes/edges with layout algorithm
- `sandbox/fprg-ast.ts` тАФ FPRG XML parser тЖТ AST, with Chinese-localized labels
- `sandbox/node-components/` тАФ Custom VueFlow node components (Start, End, Declare, Assign, Input, Output, If, Merge, For)
- `sandbox/node-components/InsertNodePanel.vue` тАФ Draggable node insertion popup (categorized by ш╛УхЕе/ш╛УхЗ║, хПШщЗП, цОзхИ╢, х╛кчОп)
- `sandbox/node-components/LayoutDebugPanel.vue` тАФ Real-time layout parameter tweaking

### Tauri Backend (`demo-f-g/src-tauri/`)
The Rust-based desktop shell that hosts the Vue frontend in a native WebView.

```
src-tauri/
тФЬтФАтФА Cargo.toml              # Rust dependencies (tauri 2, serde)
тФЬтФАтФА tauri.conf.json         # App config: window 1280├Ч800, devUrl, build hooks
тФЬтФАтФА build.rs                # tauri-build script
тФЬтФАтФА src/
тФВ   тФЬтФАтФА lib.rs              # Tauri Builder entry point
тФВ   тФФтФАтФА main.rs             # Windows entry (no console window in release)
тФЬтФАтФА capabilities/
тФВ   тФФтФАтФА default.json        # Permissions: core:default
тФФтФАтФА icons/                  # App icons (generated by `tauri icon`)
```

**Tauri v2 key concepts:**
- **Capabilities-based permissions:** `capabilities/default.json` controls what Rust commands each window can call. `core:default` enables all core plugin defaults.
- **IPC:** Frontend calls Rust via `@tauri-apps/api/core` (`invoke`), Rust calls frontend via `app_handle.emit()` (events).
- **CLI and API are separate:** `@tauri-apps/cli` is dev/build tooling; `@tauri-apps/api` is the JS runtime library.
- **Plugins:** Non-core features (dialog, fs, shell, http, etc.) are now Tauri plugins тАФ install separately via `@tauri-apps/plugin-<name>`.
- **Vite config:** Must bind to `127.0.0.1:1420` (not `localhost` which resolves to IPv6 `::1`). Use `TAURI_DEV_HOST` env var for mobile dev.

## Notes

- The library fires `flowgorithmjs:preload` and `flowgorithmjs:postload` jQuery events on the output element for extensibility
- SVG download functionality is built into the rendered output
- Functions (subroutines) beyond `Main` are supported with proper START/RETURN endpoints
- Tauri dev uses `beforeDevCommand: "pnpm dev"` тАФ the Vite server must be running before the Rust backend starts
- For new terminal sessions, ensure `%USERPROFILE%\.cargo\bin` is in PATH for Rust toolchain access
- Edge hover CSS uses `stroke` color change only (no `filter`) to avoid Chromium GPU compositing layer flicker
- FlowEdge type uses `EdgeMarkerType` from `@vue-flow/core` and `markerEnd: { type: 'arrowclosed', color: 'context-stroke' }` for SVG2 dynamic arrow coloring
